# Regulations Reference

This reference documents all regulations included in the Data Residency & Sovereignty framework.

## EU Regulations

### SecNumCloud 3.2 (France)

| Field | Value |
|-------|-------|
| ID | `FR-SECNUMCLOUD-3.2` |
| Short Name | SecNumCloud 3.2 |
| Jurisdiction | France (FR) |
| Status | Enforceable |
| Effective Date | 2022-03-01 |

French national security certification for cloud providers. Key requirements:

- **24/39 Ownership Rule**: At least 24% must be EU-owned, no single non-EU entity over 39%
- **Immunity from Extraterritorial Laws**: Provider must not be subject to US CLOUD Act
- **Data Localization**: Data must remain in France/EU

### NIS2 Directive

| Field | Value |
|-------|-------|
| ID | `EU-NIS2` |
| Short Name | NIS2 |
| Jurisdiction | European Union (EU) |
| Status | Enforceable |
| Effective Date | 2024-10-17 |

EU directive on security of network and information systems. Key requirements:

- **Supply Chain Security**: Article 21 requires vetting of ICT suppliers
- **Essential Entities**: Stricter requirements for critical infrastructure
- **Incident Reporting**: 24-hour early warning, 72-hour incident notification

### DORA

| Field | Value |
|-------|-------|
| ID | `EU-DORA` |
| Short Name | DORA |
| Jurisdiction | European Union (EU) |
| Status | Enforceable |
| Effective Date | 2025-01-17 |

Digital Operational Resilience Act for financial services. Key requirements:

- **ICT Third-Party Risk**: Article 28 requires due diligence on cloud providers
- **Concentration Risk**: Avoid over-reliance on single providers
- **Exit Strategies**: Must have documented exit plans from critical providers

### GDPR

| Field | Value |
|-------|-------|
| ID | `EU-GDPR` |
| Short Name | GDPR |
| Jurisdiction | European Union (EU) |
| Status | Enforceable |
| Effective Date | 2018-05-25 |

General Data Protection Regulation. Key requirements:

- **Lawful Processing**: Valid legal basis for personal data processing
- **Data Subject Rights**: Access, rectification, erasure, portability
- **Cross-Border Transfers**: Adequate safeguards for non-EU transfers

### Schrems II

| Field | Value |
|-------|-------|
| ID | `EU-SCHREMS-II` |
| Short Name | Schrems II |
| Jurisdiction | European Union (EU) |
| Status | Enforceable |
| Effective Date | 2020-07-16 |

ECJ judgment on US data transfers. Key implications:

- **Privacy Shield Invalidated**: Cannot rely on Privacy Shield for US transfers
- **SCCs Insufficient Alone**: Standard Contractual Clauses need supplementary measures
- **US Surveillance Risk**: Must assess US government access risk

EU regulation on data access and cloud switching. Key requirements:

## Saudi Arabia Regulations

### PDPL

| Field | Value |
|-------|-------|
| ID | `KSA-PDPL` |
| Short Name | PDPL |
| Jurisdiction | Saudi Arabia (KSA) |
